## Documentation Standards (Only When Requested)
**Core Philosophy**: User-centric, task-oriented (not feature-oriented).

### Style Reference (Standard Markdown)
Use standard markdown for compatibility. These patterns can be converted to mkdocs format later:

- **Options/Tabs**: Use `**Option Name:**` followed by content
- **Tips**: Use `> **Tip:** content`
- **Notes**: Use `> **Note:** content`
- **Warnings**: Use `> **Warning:** content`
- **Tables**: Use standard markdown tables `| col1 | col2 |`
- **Code blocks**: Use triple backticks with language identifier

### Structure & Template
- **Pyramid Principle**: Start with "What/Why" (1-2 sentences) -> Key info -> Details. (Ref: [LangSmith](https://docs.langchain.com/langsmith/llm-as-judge))
- **Recommended Content Flow** (Mental model, not strict headers):
  1. **Concept**: Clarify the goal (What/Why).
  2. **Scenario**: Define applicability (When to use).
  3. **Execution**: Clear, actionable steps.
  4. **Demonstration**: Complete examples.
  5. **Navigation**: Links to next steps.

### Writing Style & Formatting
- **Conciseness**: Short paragraphs (1-2 sentences). Use bullet lists.
- **Visuals**: **Bold** key terms. Use icons/callouts. Add internal links for references.
- **Action-Oriented**: Use verb headings (Create, Run, Analyze). Focus on user goals.
